1  Ollama

1.1 Ollama 简介

“Get up and running with large language models locally.”

Ollama 是一个开源的本地大型语言模型运行与管理工具，旨在帮助开发者和终端用户在本地机器上快速运行、下载、管理和调用各类开源大语言模型（LLM），而无需持续依赖云服务。Ollama 的定位是“在本地启动并运行大型语言模型”，持续简化本地 LLM 的落地和应用开发流程。

1.1.1 Ollama 为什么要存在

Ollama 的存在，并不是因为“本地跑模型很酷”，而是因为 云端大模型在现实世界中存在结构性缺陷，而这些缺陷不是通过“更便宜的 API”就能解决的。

1. 云端 LLM 的四个根本约束

在 2023–2025 年，大模型应用快速普及，但同时暴露出一组不可回避的问题：

  • 成本不可控
    API 调用是OPEX模式，且与使用规模强绑定，一旦进入真实业务或高频场景，成本呈指数级增长。

  • 隐私与数据主权问题
    医疗、法律、企业内部资料、个人笔记等数据，天然不适合外传至第三方云。

  • 可控性与稳定性不足
    云模型的版本升级、策略调整、限流、封禁，对开发者来说是外生风险。

  • 离线 / 边缘场景缺位
    本地开发、内网环境、边缘设备、弱网环境，云方案本身不可达。

Ollama 的存在，本质上是对这些问题的一次系统性回应。

2. Ollama 的核心定位

Ollama 是一个“把大模型变成一种本地基础设施能力”的工具。

它试图完成的不是“替代云模型”，而是让 LLM 像数据库、像 Redis、像 Docker 一样，成为开发者可以本地掌控的基础设施组件。

1.1.2 Ollama 的结构

如果从工程与系统角度看，Ollama 并不是一个“模型集合”，而是一个 分层明确的本地 LLM 运行平台。

1. 核心结构拆解

（1）模型运行与推理引擎层

  • 基于 llama.cpp 生态 与 GGUF 模型格式

  • 统一处理：
    CPU / GPU（Metal / CUDA / ROCm）调度
    量化模型加载
    推理上下文管理

这一层解决的是：“模型能不能在这台机器上跑起来”。

（2）模型管理与分发层

  • ollama pull / run / list / rm

  • 模型版本、参数、量化形式的统一管理

  • 官方模型仓库 + 社区模型

这一层解决的是：“模型作为资源如何被管理”。

（3）服务接口层（API / CLI）

  • CLI：面向开发者与技术用户

  • REST API：
    /api/generate

    /api/chat

    /api/embeddings

这一层解决的是：“模型如何被程序调用”。

（4）用户体验与生态层

  • GUI 客户端（2025 年后逐步成熟）

  • 与 LangChain、LlamaIndex、Continue、IDE 插件等集成

  • 本地 Agent / RAG / 文档分析工具链

这一层解决的是：“模型如何进入真实工作流”。

2. 一个关键理解

Ollama 不是模型本身，也不是“又一个 AI 应用”，而是LLM Runtime + Model Manager + Local AI Service

（1）本地开发与实验环境（Developer First）

Prompt / Agent / RAG 快速实验、无 API Key、无费用焦虑、可调试、可复现

1.1.4 关于 Ollama 的常见误解

误解 1：Ollama ≈ 一个模型 / 一个聊天工具

Ollama 不是模型，也不是 ChatGPT 的本地替代品。

它是模型运行平台，就像 Docker不是应用本身。

误解 2：本地模型一定比云模型“差很多”

在大量任务中，比如文档理解、总结、分类、RAG问答，中等规模模型已经“够用且更可控”。

很多场景需要的是 可靠性与一致性，不是榜单分数。

误解 3：Ollama 只适合“极客玩具”

事实恰恰相反，GUI 的出现、工具链的成熟、Agent/RAG的落地使 Ollama 正在从极客工具 → 工程工具 → 生产工具 迁移。

误解 4：用了 Ollama 就不需要云模型了

这是一个非此即彼的误解。现实中更常见的是本地 Ollama + 云模型混合架构，本地负责隐私 / 高频 / 成本敏感，云端负责复杂推理 / 超大上下文 / 多模态。

1.2 Ollama 特点

  • 本地可控，而非只是“免费”

    Ollama 的核心价值是把 LLM 变成你本地可掌控的基础设施：模型版本、参数、更新节奏都由你决定，而不是被云服务绑定。

  • 真正的一键式本地 LLM Runtime

    无需理解 CUDA、显存、量化细节，通过简单命令即可完成模型下载、运行和切换，复杂性被封装在平台内部。

  • 跨平台且工程级稳定

    稳定支持 macOS、Linux、Windows，并可通过 Docker 部署，已从“能跑”进化到“可长期使用”。

  • 模型生态成熟、覆盖主流开源路线

    支持并持续更新主流开源模型（如 LLaMA、Qwen、Gemma、DeepSeek 等），适配不同规模与场景需求。

  • 既是工具，也是本地 AI 服务层

    提供统一 CLI 与 REST API，可作为本地 Copilot、RAG、Agent、批量推理等应用的后端服务。

  • 隐私与成本结构天然友好

    数据不出本地、零 API 调用成本，特别适合高频调用、隐私敏感或内网/离线场景。

  • 常见误解澄清

    Ollama 不是“聊天软件”，也不是“云模型替代品”，而是本地 LLM 运行与管理平台，常与云模型形成混合架构。

1.3 支持的模型

Ollama 支持的模型库列表 https://ollama.com/library。

ollama serve 启动 Ollama
ollama create 从 Modelfile 创建模型
ollama show 显示模型信息
ollama run 运行模型
ollama stop 停止正在运行的模型
ollama pull 从注册表中拉取模型
ollama push 将模型推送到注册表
ollama sign in 登陆ollama.com
ollama sign out 登出ollama.com
ollama list 列出所有模型
ollama ps 列出正在运行的模型
ollama cp 复制模型
ollama rm 删除模型
ollama help 显示任意命令的帮助信息
标志 描述
-h, --help 显示 Ollama 的帮助信息
-v, --version 显示版本信息

终止 Ollama 模型推理服务，可以使用 /bye。

注意：Ollama 进程会一直运行，如果需要终止 Ollama 所有相关进程，可以使用以下命令：

Get-Process | Where-Object {$_.ProcessName -like '*ollama*'} | Stop-Process

它的作用是查找并强制停止所有名称中包含 “ollama” 的正在运行的进程。

这条命令通常用于：

  1. Ollama 卡死 / 无响应
  2. 端口被占用，无法重新启动
  3. Windows 下 Ollama 后台服务异常
  4. 需要“彻底重启” Ollama
